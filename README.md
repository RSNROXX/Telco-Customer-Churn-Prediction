## üìå Project Overview
Customer Churn (attrition) is a critical metric for subscription-based businesses. This project builds a **Machine Learning pipeline** to predict which customers are at risk of leaving a telecommunications company.

Unlike standard prediction models, this project focuses on **interpretability** and **business value**:
1.  **Recall Optimization:** We prioritize identifying churners (Class 1) to minimize revenue loss.
2.  **"The Why":** We use Logistic Regression Odds Ratios to explain *why* customers leave.
3.  **Efficiency:** We use **Recursive Feature Elimination (RFE)** to find the top 10 drivers of churn.

## üß† Key Techniques Used
 - Logistic Regression: Selected for its ability to provide probability scores and interpretable coefficients.
 - Class Weight Balancing: Applied class_weight='balanced' to handle the dataset imbalance (Non-Churners > Churners), significantly improving the model's sensitivity.
 - Recursive Feature Elimination (RFE): Automated feature selection to identify the top 10 most impactful features, reducing noise and overfitting.
 - Probability Thresholding: The system outputs exact churn probabilities, allowing for tiered retention strategies.

## ‚öôÔ∏è Project Workflow

    - Standard Scaling for numerical features.
3. EDA & Baseline: Establishing a performance benchmark.
4. Model Training: Training a Full Logistic Regression model with balanced weights.
5. Feature Selection: Reducing dimensionality using RFE.
6. Evaluation: Analyzing the Confusion Matrix and ROC-AUC Curve.
7. Deployment Simulation: A manual interface to input new customer data and get real-time risk predictions.

## üìä Key Business Insights

 - üî¥ High Risk: Customers with Fiber Optic internet and Electronic Check payments are most likely to churn.
 - üü¢ Retention Drivers: Two-Year Contracts and Tenure (loyalty) significantly reduce the probability of churn.
